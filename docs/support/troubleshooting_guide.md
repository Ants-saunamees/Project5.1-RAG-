# Support Troubleshooting Guide

This document provides a complete reference for diagnosing and resolving common customer issues. Support agents must follow these steps before escalating to engineering.

## 1. Login Issues
Most login problems are caused by incorrect passwords, expired sessions, or disabled accounts. Agents must verify the customer’s email, check account status in the admin panel, and confirm whether the user recently changed devices. If the issue persists, reset the session token.

## 2. Payment Failures
Payment failures typically occur due to insufficient funds, expired cards, or bank declines. Agents must check the payment logs, verify the card expiration date, and confirm whether the customer’s bank flagged the transaction. If the issue is internal, escalate to Billing Support.

## 3. Slow Performance
Slow performance may be caused by network issues, outdated app versions, or server load. Agents must ask the customer for device type, OS version, and network conditions. If multiple users report the same issue, notify the on‑call engineer.

## 4. Data Sync Problems
Sync issues often occur when the customer uses multiple devices. Agents must confirm the last sync timestamp and check the sync logs. If the logs show repeated failures, escalate to engineering.

## 5. Escalation Rules
Agents may escalate only after completing all required troubleshooting steps. Escalations must include logs, timestamps, and a summary of actions taken.

This guide ensures consistent and efficient customer support.
